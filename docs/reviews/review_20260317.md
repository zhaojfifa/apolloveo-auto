## 三、业务逻辑洞察

### 3.1 Hot Follow 四层状态模型

本次审查发现 Hot Follow 的状态实际由**四层语义**叠加构成，这是理解所有状态 bug 的根基：

| 层 | 语义 | 来源 | 示例 |
|----|------|------|------|
| Layer 1 | **Pipeline Step Status** | 各步骤执行结果 | `parse=done, subtitles=running, dub=pending` |
| Layer 2 | **Artifact Existence** | 产物文件是否存在 | `final_exists=true, audio_exists=true` |
| Layer 3 | **Currentness / Resolution** | 产物是否与当前版本匹配 | `dub_current=false`（字幕改了但配音没重跑） |
| Layer 4 | **Ready Gate** | 复合判断：是否可合成/发布 | `subtitle_ready=true, audio_ready=false, compose_ready=false` |

**核心问题**：前端 Workbench 和后端 Hub 在不同位置使用不同层的数据，导致：
- Pipeline bar 用 Layer 1 显示 pending
- Quick bar 用 Layer 4 显示 ✓（因为 `no_dub=true` 绕过了 audio 门控）
- 两者矛盾 → 运营困惑

### 3.2 内容路由：双通道状态机

`_hf_dual_channel_state()` 是 Hot Follow 的内容路由核心，判断三种模式：

| content_mode | 含义 | 配音策略 | 合成策略 |
|---|---|---|---|
| `voice_led` | 标准口播（有稳定人声） | 正常配音 | 配音+字幕 |
| `silent_candidate` | 无人声/ASMR | `no_dub=true` | 字幕版 |
| `subtitle_led` | 画面文字/BGM 驱动 | `no_dub=true` | 字幕+BGM |

**关键发现**：该函数用 `has_text`（字幕 lane 是否有文本）判断 `speech_detected`。
当字幕步骤仍在 `running` 时，文本为空不代表无语音——只是 ASR 还没跑完。
这是多个状态不一致 bug 的**根因**。

### 3.3 Compose 流程

合成端点 `compose_hot_follow_final_video()` 是全系统最复杂的单一函数，包含：
- DB 级 advisory lock（`compose_lock_until`）
- Revision snapshot 对比（防止 stale 提交）
- FFmpeg 合成（字幕烧录 + 配音混合 + freeze-tail + 清洁模式）
- R2 上传
- policy_upsert 状态更新
- 多层 exception handler（HTTPException / Exception / finally）

### 3.4 Workbench Hub（数据聚合中心）

`get_hot_follow_workbench_hub()` 是 workbench 页面的唯一数据源，聚合：
- Pipeline steps 状态
- Audio/subtitle/compose 诊断信息
- Ready gate
- Deliverables 列表
- Ops guide（任务判断卡）
- Voice options（TTS 声线列表）
- Translation QA（翻译质量检查）

该端点约 350 行，是理解 workbench 行为的关键。

## 四、隐患排查与已修复 Bug

### 🔴 P0 — 已修复

#### Bug 1: `_hf_dual_channel_state()` 在字幕 running 时误判 `silent_candidate`

**根因**: 字幕步骤仍在 running → 文本为空 → `has_text=false` → `speech_detected=false` → `content_mode=silent_candidate` → `no_dub=true`

**影响**:
- Pipeline bar 配音 dot 显示 pending（Layer 1 正确）
- Quick bar 配音显示 ✓（Layer 4 被 `no_dub=true` 误导）
- 任务判断卡显示"无人声素材"，但视频实际有人声
- 运营被误导，无法走标准配音路径

**修复**: 增加 `subtitles_step_done: bool` 参数，当字幕未完成时默认 `voice_led`
```python
subtitle_still_pending = not subtitles_step_done and not has_text and not no_subtitles
if subtitle_still_pending:
    content_mode = "voice_led"  # 不确定时默认走标准路径
    speech_detected = True
    speech_confidence = "pending"
```
**文件**: `gateway/app/routers/tasks.py:1957`
**Commits**: 76e15a3, 后续修复

---

#### Bug 2: Compose 500 级联错误 — `datetime` UnboundLocalError

**根因**: `compose_hot_follow_final_video()` 函数内部有一处 `from datetime import datetime, timezone`（在 advisory lock 检查的 try 块内，line 5641）。Python 将 `datetime` 视为**整个函数作用域的局部变量**。当 except handler 在该行执行前调用 `datetime.now(timezone.utc)` 时，触发 `UnboundLocalError`。

**影响**: 所有 compose 提交返回 500 / 409，错误信息：`"cannot access local variable 'datetime' where it is not associated with a value"`

**修复**: 将局部 import 重命名为 `_dt_cls` / `_tz`，不再遮蔽模块级 `datetime`
```python
from datetime import datetime as _dt_cls, timezone as _tz
```
**文件**: `gateway/app/routers/tasks.py:5641`
**Commit**: bb73f5d

---

#### Bug 3: Compose exception handler 级联失败

**根因**: except / finally 块中的 `_policy_upsert` 调用本身可能抛异常，掩盖原始错误

**修复**: 每个 `_policy_upsert` 和 `lock.release()` 独立包裹 try/except，失败时静默记录日志
```python
except HTTPException as exc:
    try:
        _policy_upsert(repo, task_id, task, {...}, step="compose")
    except Exception:
        logger.exception("compose: policy_upsert in HTTPException handler failed")
    raise
```
**文件**: `gateway/app/routers/tasks.py:5748-5788`
**Commit**: 76e15a3

---

#### Bug 4: `no_dub` 传播路径错误

**根因**: `payload.get("no_dub")` 在赋值前被读取（`no_dub` 此时存储在 `payload["audio"]["no_dub"]` 中）

**修复**:
```python
# Before:
no_dub = bool(payload.get("no_dub"))
# After:
no_dub = bool((payload.get("audio") or {}).get("no_dub"))
```
**文件**: `gateway/app/routers/tasks.py:4663`

## 五、运营现状分析

### 5.1 已形成的运营能力

1. **完整主链闭环**: 输入视频 URL → 最终成片下载 → 已在 workbench 截图确认
2. **多内容模式识别**: voice_led / silent_candidate / subtitle_led 自动路由
3. **运营可观察性雏形**: 任务判断卡 + pipeline bar + quick bar + 诊断面板
4. **辅助输入工具链**: OCR 候选 → 辅助输入区 → 翻译 → 字幕编辑区
5. **Azure Speech 稳定 baseline**: 缅语配音已形成可复用路线

### 5.2 当前最大运营风险

**风险转移**：最大心智风险已从"链路是否成立"转移到"状态和产物如何解释"。

具体表现：
- 当前尝试失败 vs 上次成片仍存在 → 运营误解为"整个任务失败"
- ready gate 与 deliverables 不一致 → 运营不知道该点哪个按钮
- "再配音失败"不再是系统性黑洞，而是拆成了两类问题：
  - rerun 意图是否成功传递到底层执行链
  - 当前 attempt 与历史成功产物是否被正确区分
